# Система детекции нарушений СИЗ (PPE Detection)

Проект для автоматического обнаружения нарушений использования средств индивидуальной защиты (СИЗ) - касок и сигнальных жилетов - с использованием моделей компьютерного зрения YOLOv8.

## 📋 Описание

Система анализирует видеопоток или изображения с камер наблюдения и выявляет работников, не использующих необходимые средства защиты:
- **Helmet** (Каска)
- **Vest** (Сигнальный жилет)

## 🏗️ Структура проекта

```
├── modelsYolo/              # Модели YOLO
│   ├── best.pt             # Обученная модель для детекции СИЗ
│   ├── yolov8n.pt          # Базовая модель YOLOv8 Nano
│   └── yolov8n-pose.pt     # Модель для определения позы человека
├── helmetDetection/        # Результаты обучения
│   └── detect/
│       └── ppe_detection/
│           └── yolov8n_helmet_v1/  # Эксперимент обучения
├── test/                   # Тестовые изображения
├── t.py                    # Скрипт быстрой проверки модели
├── train.py                # Скрипт обучения модели
├── test_tracing.py         # Основной скрипт обработки видео/изображений
└── violations_log.json     # Лог зафиксированных нарушений
```

## ⚙️ Требования

- Python 3.8+
- ultralytics
- opencv-python
- numpy

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install ultralytics opencv-python numpy
```

### 2. Запуск детекции на изображении

```bash
python t.py
```

Скрипт загружает изображение из `test/TEST.jpg` и отображает результаты детекции.

### 3. Обучение модели

Для обучения собственной модели отредактируйте `train.py` и запустите:

```bash
python train.py
```

Параметры обучения:
- Модель: `yolov8n.pt` (Nano версия для быстродействия)
- Эпохи: 100
- Размер изображения: 640x640
- Устройство: CPU/GPU

### 4. Обработка видео/потока

Основной скрипт `test_tracing.py` поддерживает:
- Обработку видеофайлов
- Работу с камерами
- Логирование нарушений в JSON
- Сохранение кадров с нарушениями

## 📊 Формат логов нарушений

Нарушения сохраняются в `violations_log.json`:

```json
{
  "timestamp": "2026-04-15T22:55:07.604845",
  "camera_id": "entrance",
  "violation_type": "helmet_vest",
  "person_idx": 0,
  "confidence": 0.63,
  "present_classes": []
}
```

Типы нарушений:
- `helmet` - отсутствует каска
- `vest` - отсутствует жилет
- `helmet_vest` - отсутствуют оба средства защиты

## 🎯 Классы объектов

Модель распознаёт следующие классы:
- `person` / `human` - человек
- `helmet` - каска
- `vest` - сигнальный жилет

## 📁 Результаты обучения

Результаты обучения сохраняются в директорию `helmetDetection/detect/ppe_detection/`:
- `args.yaml` - параметры обучения
- `results.csv` - метрики по эпохам
- `labels.jpg` - визуализация размеченных данных
- `train_batch*.jpg` - примеры батчей обучения
- `weights/` - веса модели

## 🔧 Настройка

### Параметры детекции

В скриптах можно настроить:
- `conf` - порог уверенности (по умолчанию 0.5)
- `imgsz` - размер входного изображения
- `device` - устройство для инференса (cpu/cuda)

### Добавление новых камер

Для обработки с разных камер укажите уникальный `camera_id` в настройках скрипта обработки.
